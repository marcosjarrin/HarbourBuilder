Sintaxe xBase

// Definir ListView com 3 colunas
@ 50, 20 LISTVIEW oLV OF oForm SIZE 480, 250 ;
   COLUMNS "Nome", "Idade", "Cidade"

oLV:AddItem( { "Alice",   "30", "New York"    } )
oLV:AddItem( { "Bob",     "25", "Los Angeles" } )
oLV:AddItem( { "Charlie", "40", "Chicago"     } )

Editar no Inspector

Ao selecionar um controle TListView no designer, o inspector expoe duas propriedades editaveis:

  • aColumns — clicar no botao ... abre o editor de array. Um cabecalho de coluna por linha.
  • aItems — clicar no botao ... abre o ListView Items Editor dedicado: uma grade com as colunas de aColumns, uma linha por item, duplo clique em qualquer celula para edita-la no local. Botoes + Add / - Del / Up / Down gerenciam linhas. Enter confirma, Esc cancela.

O gerador de codigo emite o comando @ ... LISTVIEW ... COLUMNS ... seguido de uma chamada :AddItem({...}) por linha.

Formato de armazenamento

Internamente aColumns e aItems sao serializados como strings:

  • aColumns: cabecalhos separados por | — ex. "Nome|Idade|Cidade"
  • aItems: linhas separadas por |, celulas dentro de cada linha por ; — ex. "Alice;30;NY|Bob;25;LA"

Modos de exibicao

A propriedade nViewStyle aparece no inspector como dropdown com os valores enumerados (vsIcon, vsList, vsReport, vsSmallIcon). Alterar o valor em runtime atualiza o controle imediatamente. Os modos Icon e SmallIcon requerem um ImageList associado para exibir icones.

Editar imagens no Inspector

A propriedade aImages abre um dialogo dedicado ListView Images Editor ao clicar ...: lista de caminhos + botoes + Add (file picker .png/.ico/.bmp), - Del, Up, Down. As imagens carregam em runtime via GDI+, escaladas para 32x32 (vsIcon) e 16x16 (vsSmallIcon). Atribuicao por linha e round-robin: row[N]:icone = aImages[N mod len(aImages)].

// Definir ListView com icones
@ 50, 20 LISTVIEW oLV OF oForm SIZE 480, 250 ;
   COLUMNS "Nome", "Idade" ;
   ITEMS "Alice;30", "Bob;25" ;
   IMAGES "user.png", "admin.png"

oLV:nViewStyle := 0  // Modo Icon: exibe PNGs 32x32
