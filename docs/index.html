The pitch

xBase meets C++Builder-style productivity.

HarbourBuilder is a modern visual IDE that generates Harbour/xBase code. The designer, the inspector and the editor stay in sync — edit one and the others update instantly. Forms are just Harbour classes; the code is yours.

What you write

// Form1.prg
#include "hbbuilder.ch"

function Main()
   local oForm, oBtn

   DEFINE FORM oForm ;
      TITLE "Hello World" ;
      SIZE 400, 300

   @ 120, 140 BUTTON oBtn ;
      PROMPT "Click Me!" ;
      OF oForm SIZE 120, 32
   oBtn:OnClick := { || MsgInfo( "Hi!" ) }

   ACTIVATE FORM oForm CENTERED
return nil

What the IDE generates

CLASS TForm1 FROM TForm
   DATA oBtn1  // TButton
   METHOD CreateForm()
ENDCLASS

METHOD CreateForm() CLASS TForm1
   ::Title := "Hello World"
   ::Width := 400
   ::Height := 300

   @ 120, 140 BUTTON ::oBtn1 ;
      PROMPT "Click Me!" ;
      OF Self SIZE 120, 32
return nil
Features

Everything a modern Harbour IDE should have.

From drag-and-drop design to AI pair-programming, all natively integrated.

🎨

Visual Designer

130 controls across 16 palette tabs. Snap-to-grid, rubber-band selection, resize handles, two-way sync with code.

🔍

Object Inspector

Properties, events, and component tree. Specialized editors for colors, fonts, files, arrays, hierarchies.

📝

Scintilla Editor

Harbour syntax highlighting, tabs, find/replace dialog, autocomplete, bracket matching, dark theme.

🐞

Integrated Debugger

Watch, call stack, locals, breakpoints, step in/over/out — all inside the IDE, no external tools.

↔️

Two-Way Sync

The designer updates code; the code updates the designer. Live, debounced, loss-free. Works both ways.

🖥️

100% Native UI

Win32, Cocoa, GTK3 backends. Real native controls on each platform — not themed widgets or a browser.

🗄️

Data-Aware

TBrowse, DBGrid, DBNavigator. DBF, MySQL, MariaDB, PostgreSQL, SQLite, Firebird, SQL Server, Oracle, MongoDB.

🤖

AI Assistant

OpenAI, Gemini, Claude, DeepSeek, Grok, Ollama, Whisper, Embeddings, Transformers — all as drop-in components.

🔌

Language Interop

TPython (in-process via dlopen), TNode, TLua, TRuby, TGo, TRust, TSwift, TJava, TDotNet.

How it works

The same architecture that made HarbourBuilder native on three desktops — now extended to two more.

Your .prg  ──  DEFINE FORM ... BUTTON ...
    │
    ▼
classes.prg  ──  calls UI_FormNew(), UI_ButtonNew() ...
    │
    ├──▶  hbbridge.cpp   →  Win32   (CreateWindowEx)
    ├──▶  cocoa_core.m   →  macOS   (NSView hierarchy)
    ├──▶  gtk3_core.c    →  Linux   (GtkWidget)
    ├──▶  android_core.c →  Android (JNI → android.widget.*)  ◀── NEW
    └──▶  ios_core.m     →  iOS     (UIKit / UIView)          ◀── SOON
📲

Native widgets

No web view. No skinned UI. Real android.widget.Button, real material feel.

🎨

Density-aware

Form-designer coordinates scale by DisplayMetrics.density — consistent on mdpi, hdpi, xxhdpi.

⚡

One-click build

Menu Run → Run on Android. 8-step pipeline: harbour → clang → aapt2 → d8 → apksigner. Fully automated.

🔁

Write once

The same .prg that targets Windows, macOS and Linux now also produces a signed APK. iOS next.

Get started

Download & build.

Clone the repo and build from source. Pre-built binaries are distributed via GitHub Releases.

🪟

Windows

MSYS2 / MinGW build. Ships a native Win32 IDE.

build_win.bat
🍎

macOS

Universal Cocoa build. Apple Silicon + Intel.

./build_mac.sh
🐧

Linux

GTK3 build. Tested on Ubuntu, Fedora, Arch.

./build_linux.sh

Clone it in 10 seconds.

The full source is MIT-licensed. Build it, fork it, ship it.

$ git clone https://github.com/FiveTechSoft/HarbourBuilder.git
$ cd HarbourBuilder
$ ./build_mac.sh   # or build_linux.sh / build_win.bat
$ ./bin/HbBuilder
