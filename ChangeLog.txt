2026-04-09 (Cross-platform fix: classes.prg linker errors on macOS/Linux)

  BUILD FIX (macOS/Linux):
  - classes.prg had unconditional EXTERNAL UI_STORECLRPANE / UI_HASHANDLE
    declarations and a W32_ErrorDialog() call, all Windows-only symbols
    implemented in source/cpp/hbbridge.cpp. Because hbbuilder_macos.prg

    into the Windows build
  - Reported by Marek Olszewski on macOS arm64 after pulling Session 33
    Windows fixes
