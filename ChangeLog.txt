hbcpp - C++ powered GUI for Harbour
=====================================

----------

  * New hbcpp_linux.prg: complete IDE equivalent to macOS version
  * App title: "hbcpp (GUI framework for Harbour)"
  * Main IDE bar: full-width window with menu bar, speedbar (toolbar),
    component palette (GtkNotebook with tabs), all in one horizontal strip
  * Component Palette: GtkNotebook with real tabs and buttons
    Tabs: Standard, Additional, Data Access, Data Controls
    Buttons with tooltips and OnSelect callback
  * Code Editor: GtkTextView with dark theme (VS Code colors #1E1E1E),
    syntax highlighting for Harbour/xBase (keywords bold blue, commands
    teal, strings orange, comments green, preprocessor purple,
    logical literals), line number gutter with Cairo drawing,
    scroll-synced gutter repaint
  * Inspector combo: GtkComboBoxText control selector at top of inspector
    (INS_ComboAdd, INS_ComboSelect, INS_ComboClear, INS_SetFormCtrl,
    INS_SetOnComboSel, INS_SetPos) - synced with design form selection
  * StatusBar: GtkStatusbar with context ID (replaces stub)
  * UI_FormSelectCtrl: real implementation for programmatic selection
  * Layout: Inspector left column, Code Editor right (background),
    Form Designer floating centered on top of editor
    Positions calculated from GTK_GetWindowBottom() for pixel-perfect alignment
  * New bridge functions: GTK_GetScreenWidth, GTK_GetScreenHeight,
    GTK_GetWindowBottom, CodeEditorCreate, CodeEditorSetText,
    CodeEditorGetText, CodeEditorDestroy
  * build_gtk.sh: accepts optional program name parameter
    ./build_gtk.sh (builds hbcpp_linux), ./build_gtk.sh test_design_gtk
  * inspector_gtk.prg: added InspectorPopulateCombo function

- macOS IDE: full Borland C++Builder-style layout with 4 independent windows
  * Renamed to hbcpp_macos (binary) / hbcpp_macos.prg (source)
  * App title: "hbcpp (GUI framework for Harbour)"
  * Main IDE bar: borderless full-width window (no title bar, no shadow)
    Toolbar with 32px buttons + draggable splitter + component palette
  * Component Palette: NSSegmentedControl tabs at bottom, 52x50 buttons
    Tabs: Standard, Additional, Data Access, Data Controls
    Real implementation replacing stubs (PalShowTab, HBSplitterView)
  * Code Editor: NSTextView with dark theme (VS Code colors), syntax
    highlighting for Harbour/xBase (keywords, commands, strings, comments,
    preprocessor, logical literals), line number gutter (HBGutterView),
    scroll-synced gutter, Cmd+Z undo, find bar
  * Inspector combo: control selection dropdown (INS_ComboAdd, INS_ComboSelect,
    INS_ComboClear, INS_SetFormCtrl, INS_SetOnComboSel, INS_SetPos)
    Synced with design form selection changes
  * Layout: Inspector left column, Code Editor right (background),
    Form Designer floating centered on top of editor
    Positions calculated from MAC_GetWindowBottom() for pixel-perfect alignment
  * Menu accelerators: explicit ACCEL parameter (not & convention)
    Standard macOS shortcuts: Cmd+N/O/S/Z/X/C/V/F/H/R/Q
  * New bridge functions: MAC_GetScreenWidth, MAC_GetScreenHeight,
    MAC_GetWindowBottom, MAC_MsgBox
  * Fixed compilation: forward declarations for HBToolBar, PALDATA,
    HBSplitterView; replaced LONG_PTR with __bridge ARC casts;
    moved MAX_MENUITEMS/MAX_TOOLBTNS defines before first use
  * commands.ch: MENUITEM now supports optional ACCEL clause
  * classes.prg: TMenuPopup:AddItem accepts optional cAccel parameter

- IDE main window with C++Builder-like layout (3 independent windows)
  * Main IDE bar: thin top strip with menu bar, speedbar (toolbar),
    component palette (TabControl with Standard/Additional tabs),
    and status bar (position, Modified, form dimensions)
  * Object Inspector: separate floating tool window positioned at left
  * Design Form: separate floating window positioned at right
  * All 3 windows are independent top-level windows sharing one message loop
  * TForm:Show() creates/shows window without entering message loop
  * TForm:Activate() enters the message loop (main window only)
  * FMainWindow flag: only main window posts WM_QUIT on close
  * Secondary windows hide on close instead of destroying

- Added TToolBar control with compact speedbar layout
  * Win32 TOOLBARCLASSNAME with CCS_NOPARENTALIGN | CCS_NORESIZE
  * Auto-sized to content width (not full form width)
  * Text buttons with tooltips: New, Open, Save, Cut, Copy, Paste, etc.
  * Per-button OnClick callbacks via Harbour codeblocks
  * Bridge: UI_ToolBarNew, UI_ToolBtnAdd, UI_ToolBtnAddSep, UI_ToolBtnOnClick
  * Harbour class TToolBar + xBase commands DEFINE TOOLBAR / BUTTON / SEPARATOR

- Added TComponentPalette with native TabControl
  * Win32 WC_TABCONTROL positioned right of speedbar
  * Multiple tabs (Standard, Additional) with component buttons
  * Tab change via TCN_SELCHANGE notification
  * Buttons represent control types (Label, Edit, Button, CheckBox, etc.)
  * Bridge: UI_PaletteNew, UI_PaletteAddTab, UI_PaletteAddComp

- Added menu bar support to TForm
  * Win32 HMENU with popup menus and menu items
  * Per-item OnClick callbacks via Harbour codeblocks
  * Bridge: UI_MenuBarCreate, UI_MenuPopupAdd, UI_MenuItemAddEx, UI_MenuSepAdd
  * Harbour class TMenuPopup + xBase commands DEFINE MENUBAR/POPUP/MENUITEM

- Added StatusBar to TForm
  * Win32 STATUSCLASSNAME with 3 panels (position, status, dimensions)
  * Auto-created during CreateAllChildren when FHasStatusBar is set
  * Bridge: UI_StatusBarCreate, UI_StatusBarSetText

- Added form positioning and style properties

- Updated all 3 backends (Win32, Cocoa, GTK3)

- Design mode adjustments for toolbar offset
  * FClientTop: vertical offset below toolbar/palette
  * Grid dots start at FClientTop
  * Mouse coordinates adjusted by -FClientTop for design operations
  * SetWindowPos calls add +FClientTop for physical positioning
  * PaintSelectionHandles adds FClientTop to Y coordinates

- Added Linux GTK3 backend
  * gtk3_core.c: full reimplementation of C++ core using GTK3

  * inspector_gtk.prg: Harbour wrapper for GTK3 inspector
