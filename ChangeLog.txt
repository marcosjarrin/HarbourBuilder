=====================================

2026-04-02 (HbBuilder: Run, two-way tools, dot grid, TApplication)
-------------------------------------------------------------------
- Run button (F9): full compile+link+execute pipeline
  * Saves all .prg files to /tmp/hbbuilder_build
  * Compiles user code + framework with Harbour + clang
  * Links with Cocoa + Harbour libs, launches in Terminal.app
  * Build log with step-by-step progress, error dialog on failure
  * MAC_ShellExec: NSTask-based shell command execution
- TApplication class: Title, aForms, CreateForm (calls form's CreateForm),
  Run (activates main form)
- TForm: added Title ACCESS/ASSIGN (alias for Text, C++Builder compat)

  * RegenerateFormCode: reads live form + all children, regenerates complete code
  * SyncDesignerToCode: called on control move/resize, property edit, component drop
  * INS_SetOnPropChanged: inspector callback after any property change
  * OnComponentDrop simplified: just names the control, SyncDesignerToCode does the rest
- Design-time dot grid (classic C++Builder):
  * HBDotGridView: dedicated NSView with 8px dot pattern
  * Added to form's contentView in design mode
  * Grid snap changed from 4px to 8px (drag, resize, drop, arrow keys)

- About dialog: added "Vibe coded 100% using Claude Code"

  * Keyboard: OnKeyDown, OnKeyUp, OnKeyPress
  * Mouse: OnMouseDown, OnMouseUp, OnMouseMove, OnDblClick, OnMouseWheel

  * Events tab shows all events per control type grouped by category
  * Column headers change: Property/Value vs Event/Handler

  TButton/TEdit/TCheckBox/TComboBox/TLabel/TGroupBox with appropriate events)
- Code generation follows C++Builder OOP pattern:
  * Project1.prg with TApplication that creates TForm1
  * Form1.prg with CLASS TForm1 FROM TForm (DATAs + METHODs)
  * Empty form designer on startup (like File > New > VCL Forms Application)
- Palette icon indices: filled for all tabs (Additional 9-11, Data Access 12-13,
